# Полный импорт базы предметов

Источник каталога:

`https://corepunk.help/items?type=weapon,implant,chip,rune,consumable,resource`

Импортёр принимает только шесть разрешённых типов:

- `weapon`;
- `implant`;
- `chip`;
- `rune`;
- `consumable`;
- `resource`.

Таланты, задания, скины, транспорт и другие разделы не импортируются.

## Что сохраняется

- полные записи всех предметов из выбранного каталога;
- варианты качества Common, Uncommon, Rare и Epic для улучшаемых предметов;
- характеристики, теги, описания, эффекты, модификации, цены и требования;
- базовые и синтез-рецепты со всеми компонентами и количеством;
- локальные изображения предметов каждого качества;
- доступные иконки характеристик, в том числе используемые внутри описаний;
- иконки профессий;
- направленные связи между предметом и каждым компонентом рецепта;
- будущий маршрут карточки и ключ записи для будущего hover-превью каждого компонента.

Плитки компонентов пока намеренно не являются ссылками, а hover-превью не показывается. Данные для обеих функций уже сохранены в `relations.targets` и выведены в HTML как служебные `data-*` атрибуты.

Исходные игровые данные сохраняются на английском языке. Русский перевод выполняется отдельным этапом и не изменяет исходный снимок.

## Запуск

```powershell
pnpm scrape:items
```

Повторный запуск использует локальный кеш детальных карточек и повторно не загружает уже сохранённые изображения. Чтобы принудительно обновить данные с сайта:

```powershell
node scripts/scrape-corepunk-items.mjs --refresh
```

Итоговый набор данных сохраняется в `src/data/corepunk-items.json`, изображения — в `public/game-assets`, а технический кеш — в `scripts/.cache`.

Тестовый импорт одной карточки сохранён как вспомогательный скрипт `pnpm scrape:test-item`, но портал использует полный набор данных.
